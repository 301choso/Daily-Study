### 검증 헤더와 조건부 요청

서버에 다시 요청하면 두가지 상황 나타남

1) 서버에서 기존 데이터를 변경함

2) 변경하지 않음

시간 만료후에도 서버에서 데이터 변경하지 않음

캐시 재사용 할 수  있다. 단 클==서버 같다는 검증필요

검증헤더 추가(데이터가 마지막에 수정된 시간)

Last-Modified : 2020년 11월 10일 10:00:00

클라이언트가요청 보낼때 보냄

if-modified-since : 2020년 11월 10일 10:00:00

이를 통해 서버에서 데이터가 수정되지 않았는지 확인 가능

서버 응답 :

HTTP/1.1 301 Not Modified

Last-Modified : 2020년 11월 10일 10:00:00

바디가 없음 (헤드만 전송 -> 부하가 줄어듦)

클라이언트에서 캐시 재사용하고 다시 셋팅

---

ETag 캐시용 데이터에 임의의 고유한 버전 이름을 달아둠

(해시 알고리즘으로 만들거나 함)

데이터 변경시 이 이름을 바꾸어서 변경함(Hash 다시 생성)

단순하게 ETag 다르면 다시받기임
